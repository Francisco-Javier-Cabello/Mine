CSS Structure and Syntax

  • Each style sheet is made up of style rules containing two parts

    • Selector

    Declaration
    • Property
    • Value

  • Declarations can configure a single or multiple values

    • H1 {color:teal;}
    • H1 {font-family:Arial;}
    • H1 {color:teal; font-family:Arial;}

  • Whitespace is irrelevant but each designer will have their own preference as to how they write CSS style rules
  • Style sheets override the browsers default styling and your style declarations will alter the final appearance of the web page in the browser

    • Keep in mind that users can override your styles!

  • Keep in mind that punctuation is KEY in CSS!

    • Mind your curly brackets and semicolons as mistyping will prevent styles from being applied!
    Use a validation service like http://jigsaw.w3.org/css-validator to help find problems in your style sheets
  • Selectors

    • The universal selector(*) - matches any element and is often used for global properties

      • * {margin:0px;}
      • The most blunt selector there is, extremely powerful but not specific

    • The element type selector

      • Select a particular element such as h1,p,a,div,etc...
      • Works the same as universal but with a single element

    • ID selectors - select an element according to its ID attribute

      • ID attributes are designed to be unique within a single HTML document providing extreme levels of precision with CSS
      • To use you preface the value of the ID you want to select with # Symbol

        • #home-image {transform: rotate(-70deg);}
        • #container {background-color:orange;}

    • Class selectors - specify certain instances of an HTML markup element

      • To use you combine the HTML markup element with the global "class" attribute
      • Then write CSS style rules to target a class prefacing the class name with a (.)

        • .warning {color:red;}
        • H1.warning {color:red; font-weight:bolder;}

      • Elements are allowed to have multiple class i.e <div class="myclass1 myclass2">

    • Selecting Multiple elements

      • Multiple elements can easily be selected
      • Div, p, ul {...}

    • Selecting child elements

      • If you want to select all the <div> element or other parent-child relationship
      • Div p {...}

    • Selecting by attribute

      • You can select an element based on the specific content of an attribute
      • A[href=http://www.google.com] {...}

    • Selecting Pseudo-classes

      • Pseudo classes refer to the special state of an element
      • The most common pseudo class are those for the <a> element
        • Link (unvisited)
        • Visited
        • Hover
        • Active (when the link is clicked)
      • A:hover {...}

    • Selecting Pseudo elements

      • Refers to a specified part of an element such as the first letter or the first line of the element
      • <p> This is some text. </p>
        • You can select the letter Tusing this selector - p::first-letter {...}

    • Case insensitivity

      • For the most part CSS selectors and rules are case-insensitive
        • Div {Background-Color:GREEN;}
        • DIV {background-coloR:green;}

      • The only exception is when selecting classes and ids which ARE case-sensitive!

    • Order of Precedence

      • A very important concept that can be frustrating if you are new to CSS
      • If you arte struggling with rules not being applied check

        • Case sensitivity for classes and ids
        • Appropiate class values for elements
        • Precedence

      • Understanding the rules for precedence is key!
      • Order of Precedence - three principles to remember

        1. The more specific the selector the higher the precedence

          • Id>Class>Element
          • The more detailed the selector the higher the precedence

            div {font-size: 10px;}
            #myId {font-size: 12px;}
            .myClass {font-size: 14px;}

            <div id="myId" class="myClass">Some text</div>

        2. If no style is specified, elements inherit styles from their parent container or use the browser´s default styling
        3. All else being equal, the last declared rule wins

Using Different Kinds of Style Sheets

  • Three options for styling exist

    • Inline styles
    • Internal style sheets
    • External style sheets

  • Of the three we want to use external style sheets whenever possible
  • The only reason to use inline styles is when you truly will never need to reuse the style again and internal style sheets when a page is unique in its styling

  • External style sheets are connected to an HTML file using the <link> element

    • Attributes
      • Rel - indicates the relationship of the linked document and the document linking to it and in this case it always be rel="stylesheet"
      • Type is not required
      • Href - indicates the location of the linked stylesheet and may use an absolute or a relative link

        • You shouldn't normally use a style sheet that doesn't reside on your website because you would not be able to control look and feel
        • One exception is to use http://www.w3.org/StyleSheets/Core to link to the standards style sheets in order to quickly add style to your web page to test styles in different browsers

  • Another method for including style sheets in your HTML document is with the @import statement

    • Instruct the browser to load an external style sheet and use its styles
    • <style> @import http://www.w3.org/StyleSheets/Core/stylesheet.css </style>

  • Minimize complexity and link as few style sheets as possible which wil aldo imporve browser performance

Margin and Padding

  • The padding controls the free space around the content inside each box while the margin controls the free space around the outside of the border of the box

  • Both of these controls are transparent and therefore changing their color is not possible however, the width can be modified to adjust the positioning of elements

  • Several syntax options exist for both margins and padding

  • Margin and Padding syntax

    • Margin: 25px; sets all four margins to 25px
    • Margin: 25px 50px; sets the top and bottom margins to 25px and the left and right margins to 50px
    • {Margin-top: 25px; margin-right: 50px; margin-bottom: 50px; margin-left: 10px;}
    • Margin: 25px 50px 50px 10px (goes clockwise starting from top)

  • Additionally margins (not padding) can have negative values

MODULE 3: WORKING WITH PAGE LAYOUTS

Positioning Elements

  • Positioning elements

    • As the name implies the CSS positioning property is used to configure the location of the element on the page

    In the case of an overlap the positioning element specifies which element sould be on top

    Four separate methods exist for positioning elements

  1. Static positioning

    • Elements are positioned according to the normal flow of the page
    • All HTML elements use static positioning by default
    • There generally isn't a reason to specify this type of position unless you are attempting to override another style rule

  2. Relative positioning

    • Relative positioning is a method that positions an element "relative to its normal position"

      • Normal is the default position when no positioning rule is specified
      • Normal is also used when static positioning is in place

    • So if an element would normally be directly beneath the preceding element using relative positioning will change that position in comparison with the normal position

    • Relative position can result in elements that overlap in which case you have to use the z-index property

      • Element with the greater z-index value will be positioned in front

  3. Fixed positioning

    • Fixed positioning means that an element will always stay at its assigned location and will not move even when the page is scrolled

      • Headers
      • Social sharing buttons
      • Footers

    • When using fixed positioning you can use the top property to specify the number of pixels the element should be from the top of the page and the left property for distance from left of the page

  4. Absolute positioning

    • Absolute positioning means that an element is positioned relative to the first parent element which has a position other than static

      • If none is found it is positioned relative to the page
      • This positioning is similar to relative positioning except that the element is positioned relative to its parent instead of its normal position

    • Floating is a technique ised to arrenage elements on a page by taking them out of the normal flow

      • Float: right; - pushes the element to the right the furthest it can go

Working With Colors

  • CSS allows authors to define colors in two separate ways

    • Name

    • Number

      • RGB notation
      • Hex notation

    • There are 16 basic colors that are a part of the CSS3 specifications along with a much longer list of extended colors which are supported by today's browsers

  • When using numbers either RGB or Hex notation can be used and in both cases yoe are choosing the appropiate mixture of Red, Green and Blue

    • RGB notation

      • The first number is the amount of red, the second the amount of green, the third the amount of blue
      • RGB (0,255,0) - Pure Green or RGB(0,0,255) - pure Blue
      • The actual code would be {color: rgb(0,255,0);}

    • Use hexadecimal notation - uses 6 digits to represent color nut is the same where the first 2 digits are for red and so on

      • 00 00 00
      • FF 00 00
      • FF 00 FF
      • The actual code would be {color: #00FFFF;}

  • Colors can be redefined using two CSS properties

    • Color

      • Defines font color
      • Also used to define colors for links in various states using pseudo classes
    • Background-color or Background

      • Defines the color of background for an entire page or for particular elements on the page

Working With Backgrounds

  • There are two options for configuring the background of an element in CSS

    • Background-color
    • Background-image

  • The use of the background-color property is similar to how yoe specify the color for a border in CSS boxes
    • Use the color name
    • Use RGB notation
    • Use Hexadecimal notation

  • In order to spice things up a bit some designers utilize the background-image property instead of a standard color

  • CSS provides great flexibility with how we want the image to be displayed

  • The first step is to specify the URL to the image using the background-image property

    • Background-image:url("imagefile.jpg");

  • By default the image is placed in the top-left corner of the element and will automatically repeat itself both wertically and horizontally

  • If this is not desirable you can use the background-repeat property to modify the default setting

    • Repeat - the default setting repeats horizontally and vertically
    • Repeat-x - the image will only repeat horizontally
    • Repeat-y - the image will only repeat vertically
    • No-repeat - the image will not be repeated

  • Examples

    • Background-repeat: repeat-x;
    • Background-repeat: no-repeat;

  • By default the background image will scroll with the page but this can also be configured using the background-attachment property

    • Fixed
    • Scroll

  • The background-position property is very useful if you want to specify which area of the background image you would like to display

    • This is useful when you have an image that is larger than the element for which it is serving as a background
    • If you have an image that is 200x300 and an element for which it portion of the image (the top left portion) will display
    • In order to display a different portion of the image you would need to change the position of the image

  • The most common way is to specify the background-position value using pixels and the syntax background-position: xpos ypos

    • The first number shifts horizontally

      • Negative values shift the image to the left
      • Positive values shift the image to the right

    • The second number shifts vertically

      • Negative values shift the image to the up
      • Positive values shift the image to the down

Understanding Fonts

  • A font is a set of characters that share a similar desing

    • Times New Roman
    • Arial
    • Verdana
    • Comic Sans

  • Font properties in CSS are used to format the appearance of words and text on a page
  • The font property is concerned with how the characters look while the text property is concerned with everything else

  • Font families define the three major categories of font types with two additional types

    • Serif

    • San serif

    • Monospace

    • Cursive

    • Fantasy


  • When specifying the font-family you should always specify multiple fonts in case the user does not have the font you specify

  • The font-size property sets the size of the text and can be set in multiple ways

    • Pixels (px)
    • Em
    • Percentage (%)
    • Keywords

    Ems and percentage are the recommended methods because users can change font sizes in the browser!

  • Examples

    • Font-size: 20px;
    • Font-size: 1.5em;
    • Font-size: 150%;
    • Font-size: large;

  • Font-style property - used to specify different styles

    • Normal
    • Italic
    • Oblique

  • Font-weight property - used to specify bold text and works as the counterpart to font-style

    • Normal, bold, or bolder (some browsers may not support bolder)
    • Numbers 100-900 (above 600 is considered bold)

  • CSS2 introduced the ability to download fonts to a user's browser using the special @font-face rule

    • It was a tough beginning and the rule was removed in 2.1 and then readded in CSS3
    • Today @font-face is supproted by virtually all browsers and gives web designers far more options than ever before

  • Font file formats

    • TrueType - standard format been around since the 80s
    • OpenType - based on TrueType and supports some advanced typographical features
    • Embedded OpenType - an IE only font file format
    • Web Open Font Format (WOFF) - not a new format but a new way to package OpenType and TrueType formats

  • Steps to using additional fonts

    • Find the font

      • Purchase

      • Use open source fonts

    • Link the font using the @font-face rule

Text And Shadow Effects

  • CSS3 contains many new porperties to style, alter, customize, tweak and enhance text on the page

  • As a result of these new properties the use of images to achieve desired text effects is greatly reduced and this has a positive effect on page load time and other advantages

    • Text takes up less storage space and downloads faster
    • Screen readers and other types of alternative browsers can read text but not images
    • Search engines can read and index text

  • Shadows are a great tool for adding an illusion of depth to a web page

  • CSS3 provides two porperties that can be used to create shadows

    • Text-shadow
    • Box-shadow

  • By applying shadows in different ways you can create many types of effects

    • Drop shadows
    • Letterpress text
    • 3D text

  • Text-shadow - property used to create text shadows

  • At its simplest

    • Text-shadow: offset-x offset-y color
    • H1 {text-shadow: 0.1em 0.1em #aaaaaa;}

  • An additional optional parameter is the blur-radius which is used to make the shadow fussy and lighter

    • The larger the value the larger and lighter the blur will be

  • The box-shadow is used to create one or more shadows for an element and has a similar syntax

    • Box-shadow: offset-x offset-y color
    • Box-shadow: 4px 4px #777777;

  • Inset text is text that appears to recede into the background

    • The key is to use a text shadow that'slighter then the text background color causing the text to look like it's inset and a shdow is being cast inside the letters
    • The lighter the shadow and darker background makes the text look inset

  • Creating 3D text is the opposite of inset text in which the text appears to protrude from the page

    • The key is to use multiple text shadows together along with another CSS3 trick which is opacity
    • It can get complicated but the effect is fantastic!

  • You can also use transparency along with a new porperty called background-clip to do a much more realistic inset otherwise known as a letterpress effect

    • The most common use of the background-clip porperty is to add a background image to text which creates some stunning effects
    • This porperty specifies whether the background extends under the border of the object

MODULE 6: LINKS, LISTS AND TABLES

Working With Lists

  • Lists are powerful tools to group together similar elements and give site visitors an easy way to view groups of information

    • Lists can be created for just about any type of content

    • Lists use at list two components

      • A markup element that tells the browser a list is on the way
      • A markup element that tells the browser this particular item goes into a list

    • Three types of lists

      • Ordered lists
      • Unordered lists
      • Definition lists

    • All list types can be styled using CSS3

  • The method for styling list is the list-style-type property which allows you to set the item markers for your list

    • Item markers refer to the characters which are used to delineate list items
    • Bullets for an unordered list by default
    • Numbers for an ordered list by default

  • Using the list-style-type property for ordered lists con select the following values

    • Lower-roman - {i,ii,iii}
    • Upper-roman - {I,II,III}
    • Decimal - default
    • Decimal-leading-zero - the number with a leading zero
    • Lower-alpha
    • Lower-greek
    • Lower-alpha

  • Using the list-style-type property for unordered lists allows you to specigy the particular shape or symbol that you want to use as a bullet

    • The default marker is a disc (filled circle)
    • Other options

      • Square
      • Circle

  • In addition you can disable the marker entirely using the list-style-type property with a value of none

  • An additional option if you do not desire any of the additional markers is to use an image of your own as the marker

  • You can use the list-style-image property to specify the url of the image that you would like to use

    • List-style-image:url('myBullet.gif');

  • The list-style-position property is used to specify whether the list item markers should appear inside or outside the content flow

    • Two values are "inside" and "outside"

  • By default list items are outside and there is a certain amount of indentation

  • If this is modified and set to inside the item markers are displayed after the indentation which simply results in more indentation

Using Transforms

  • The transform property in CSS3 allows for advance management of web page elements

    • Rotation

    • Scale

    • Skew

    • Reposition

  • All of this while the margins are left intact which results in the rest of the page being left intact

  • Transforms may not be supported by every browser that accesses your site so you should check for support for implementing on your site

    • Two types of transforms exist

      • 2D

      • 3D

  • 2D Transforms operate solely on the X and Y axis that traditional web development has been confined to and all major web browsers support this type of transform

  • Types of transforms that will change the object display

    • Translate - repositions an object

    • Rotate - rotates an object

    • Scale - strechs an object in on of the available directions

    • Skew

  • All of these options are accessed through a single style property known as transform

  • In the early days additional properties were required and are still ideal for coding to ensure support

    • -moz-transform:

    • -webkit-transform:

    • -ms-transform:

    • -o-transform:

  • The translate value of the transform property will reposition the object

    • Translate will occur to the right by default

    • Translate occurs on the Xaxis by default

      • -webkit-transform:translate(90px);

      • -To be explicit you can use translateX or trasnlateY or specify both

        • webkit-transform:translate(100px, 50px);

  • The Scale property of the transform style is used to stretch or compress the elements on our page

  • Just like translate the default for scale is the equivalent of scaleXbut you can use scaleY when needed

    • Scale does not use the PX notation but instead uses a fraction amount

    • -webkit-transform:scaleX(2.0);

  • It is possible to use multiple transforms at simultaneously

  • Consider the requirement for translating the image to a different portion of the screen and then scaling it in size

    • -webkit-transform:scaleY(0.5)translateX(90px);

  • The transform property also allows us to change how elements look on the page by skewing them

    • Changing a skew is change the 90 degree angles of intersection that exost on the box containing the element

    • A skew of 0deg would therefore have no effect but any other combination on either the X or Y axos would succeed

  • In addition to 2D transforms 3D transforms are supported by most browsers currently and a part of the CSS3 specification

  • There are several steps and values which are used to achive 3D rotation

    • rotateY

    • rotateX

    • rotate

    • rotate3D (shortcut notation)

    • Perspective (changes the perspective for a 3D view)

Using Transitions

  • Transitions describe functional and gradual state canges to objects on the web and are possible with CSS3

  • Transitions are essentially a rudimentary form of animation

  • Any element on the page can have transitions applied but a common implementation is a hyperlink that appears as a tab which transitions into the page in order to be selected

  • To create a transition effect, you must specify two things

    • The CSS porperty you want to add an effect to

    • The duration of the effect

    • This allows you to change properties values smoothly over a period of time

      • div {

      •         width: 100px;

      •         height: 100px;

      •         background: red;

      •         -webkit-transition: width 2s; /* Safari */

      •         transition: width 2s;

      • }

  • Additional properties

    • Transition-timing-function - specifies the speed curve of the transition effect

      • Ease (default) specifies a slow start then fast, then end slow

      • Linear - transition same speed from start to end

      • Ease-in

      • Ease-out

      • Ease-in-out

    • Transation-delay

    • Transation-duration
