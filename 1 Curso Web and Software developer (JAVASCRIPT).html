The Structure And Content

  • JavaScript is written in plain text just like HTML and CSS but does follow a very specific syntax which developers must learn

  • JavaScript is not a compiled language but an interpreted language meaning you write the code and hand it over to the web browser and it takes care of everything

    • so no special compiler is necessary
    • Simple text files and a browser that has JavaScript enabled is all we need

  • One of the most important things to remember about JavaScript is that it is

    •   case sensitive!!!!

    • Coming from HTML and CSS even other programming languages this may not ring a bell
    • but we must remember it!

    • HTML

      • <p>This is a paragraph</P> --- works just fine, doesn't care about case though the preceding is sloopy and to be avoided for readability

    • CSS

      • Body {font-FAMILY:Arial;Font-Size:1.2em;cOLoR:blue;} - works just fine

    • JavaScript

      • Alert("Hello World"); - does not work
      • alert("Hello World"); - does work because the alert function is declared in lowercase


  • So everything that you do in JavaScript is case sensitive
    • Variables
    • Functions
    • Methods
    • Objects
    • You name it and its case sensitive!
    • If you don't get the case right you won't see any errors, your JavaScript code will simply not work

  • This is the most important rule of the syntax but not the only one

  • Syntax
    • A set of rules defining the way in which JavaScript progrmas are constructed
    • A computer program is simply a list of instructions to be executed

      • In JavaScript these are executed by the web browser
      • These individual instructions are referred to as statements

      • Statements are separated by semicolons

        • var x = 5;
        • var y = 6;
        • var z = x + y;

      • Statements are composed of multiple elements

        • Values
        • Operators
        • Expressions
        • Keywords
        • Comments

      • Two types of values exist in JavaScript statements

        • Literals - fixed values

          • Numbers - written with or without decimal points and without quotes

            • 1001
            • 10.50

          • Strings - written within single or double quotes

            • "This is a string"
            • "This is also a string'
            • "55"

          • Variables - often used to store data

            • Keyword var is used to define the variable

              • var x;
              • var x = 5;
              • var y = 6;

            • The = sign is used to assing a value

              • x = 6;

      • Operators

        • Used to assign values to variables
          • Assignment operators
            • =
        • Used to compute values
          • Arithmetic operators
            • +-*/
        • Used to evaluate values
          • Comparison operators
            • ==,!=,<,>
      • Expressions

        • A combination of values, variables, and operators, which computes to a value
        • The computation is referred to as an evaluation
        • Expressions can also contain variables

          • 15*10myValue + 25"John"+" "+ "Doe"



        • Keywords

          • Used to identify various actions to perform

            • Var - Defines a variable
            • For - marks a block of coda to be executed so long as a condition is true
            • If... else - marks a block of code to be executed based on a condition
            • Return - exits a function
            • Break - terminated a switch or a loop
            • Function - declares a function
          •   None of these keywords will be allowed to be the name of a variable, object, or function that you create and ALL are case-sensitive
          • Comments

            • Can be added to any code within a JavaScript program in order to assist with readability and debugging
            • Single lines are commented ussing //

              • //Here are some JavaScript comments
              • Should genereally precede the code that you are commenting on
              • Multiple lines are commented using  /* these are some comments */

          • The semicolon is the separator of JavaScript statements and most programs will include multiple lines of statements
            • The semicolon is the important part
            • It can often be omitted but it should never be just in case

              • var x = 5;
              • var y = 6;
              • var z = x + y;
              • document.getElementById("demo").innerHTML = z;

          • Whitespaec doesn't matter in JavaScript just like HTML and CSS

            • Add whitespace to improve readability
            • Break long code statements over multiple lines at an operator whenever possible

Working With Loops

  • Very quickly when working with programming you realize that there is a code that you want to be repeated
  • Its more efficient to tell the browser to repeat something by typing it once than to have to retype countless times
  • Looping isn't the difficult part

    • For every link in the page {//do this code}
    • For evety field in a form on the page {//do this code}
    • 200 times {//do this code}

  • Instead the difficult is when to stop, whi's keeping track?

  • One very common loop is very similar to an if statement

    • var a = 1;

    • if (a < 10) {

    •        console.log(a))

    • }

  • Just replace the keyword if with while and it will loop while a condition evaluates to true, once it evaluates to false the loop es finished

    • var a = 1;

    • while (a < 10) {

    •        console.log(a))

    • }

  • Another difference with a loop is that we must have a tracking mechanism to ensure the loop ends when it should and doesn't run infinitely

    • This is generally achieved by incrementing the variable

      • var a = 1;

      • while (a < 10) {

      •        console.log(a))

      • a++;

      • }

  • The Do/While loop is a variant of the while loop with one major difference

    • It will always execute the code block once
    • The first check to determine whether the condition is true or not comes after this initial execution

      • var text = "";

      • do {
      •        text += "<br>The number is "+ i;

      • i++;

      • }

      • while (i < 10)

      • for (i = 0; i < 5; i++) {

      •        text += "The number is "+ i + "<br>;

      • }

  • Regardless of the type or loop there are common parts

    • Set up the index
    • Check the condition
    • Increment the condition

  • The for loop does this the most efficiently as it is all completed in a single line

  • Normally a loop will continue until the condition of the loop no longer evaluates to True but there are two exceptions using JavaScript keywords

    • Break

      • A special word that jumps us out of the loop for good
      • Not used as often with For and While loops but you might encounter it

    • Continue

      • A special word that jumps back to the top of the loop
      • So it stops that particular iteration but not the loop as a whole

Introduction To The HTML Document Object Model (DOM)

  • The JavaScript code that we write does not operate in a vacuum bur instead it is attached to HTML web pages

  • What exactly is the DOM?

    • It's not a part of JavaScript
    • It's a fairly vague term

  • The web definition is "The Document Object Model is an application programming interface that defines logical structure of well-formed an XML and HTML documents"

    • Not overly helpful


  • Taking apart the statement DOM will help to understand
    • Document - is the web page itself
    • Object - refers to the objects that are created based on the HTML structure
    • Model - a consistent way to approach the various objects

    • HTML elements as objects
    • The properties of all HTML elements
    • The methods to access all HTML elements
    • The events for all HTML elements

  • The HTMLDOM is a standard for how to get, change, add or delete HTML elements

  • DOM porperties are values that you can set or change

    • A property is a value that you can retrieve or modify

      • Deleting an HTML element
