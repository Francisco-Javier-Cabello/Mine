  • Acts as the runtime environment for all .NET applications

  • Loads and executes code compiles to intermediate language (IL)

  • Also manages base services

  • Complier converts your source code to IL

  • At run time, invokes a just-in-time compiler to create native executable code


  • Takes advantage of the host processor

  • Surprisingly, JIT compiler isn't much overhead

MODULE 3: EXAMINING A .NET APPLICATION

3.1 Introducing Assemblies

  • When you compile managed code, you create an assembly

    • different from an unmanaged code EXE or DLL

    • Contains metadata in the manifest

      • Version, name, culture and security requirements
      • List of other files that belong to the assembly, if any
      • List of public types defined in the assembly
      • List of external references required to run

  • What Kind of Programs Can You Create?

  • Support for many kinds of applications

    • Visual Studio provides templates for them

      • Windows application
      • Console application
      • Class library
      • Windows control library
      • Web control library
      • ASP.NET Web site

MODULE 5: WHAT IS C#?

5.1 What is C#?

  • Wikipedia: Multi-paradigm programming language

    • Amazing, accurate buzzworld-laden description!

  • Design goals

    • Simple, modern, general-porpuse, object-oriented
    • Support for software engineering principles
    • Usable for distributed components
    • Source code portability
    • Support for internationalization
    • For both hosted and embedded systems
    • Not intended to compete with C or assembly language

  • History of C#

  • Amders Hejlsberg is principal designer and lead architect

    • Formed a team in late 1990s to build a language for the developing .NET Framework
    • C# has roots in C++ and Java, all descendants of C

  • Name came from musical notation

    • Sharp (#) is a semitone higher in pitch than the base note
    • Convention is to use a standard keyboard's number sign, #

  • Why Should You Use C#?

  • Provides a great deal of power and control

    • At a high level of abstraction

  • Much of the power is in language features

    • But much is in the .NET Framework and CLR

  • Standard C#

  • C# is a Microsoft creation

    • But in August 2000 submitted for standardization
    • Ecma International (formerly the European Computer Manufacturer's Association, ECMA)

      • ECMA-334, the C# Language Specification
      • ISO/IEC 2370:2003, Information technology -- C# Language Specification
      • ECMA-335, the Common Language Infrastructure specification (CLI)
